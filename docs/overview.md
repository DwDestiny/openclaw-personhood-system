# OpenClaw 人格化成长系统 - 方案总览

## 解决的问题

1. **长期记忆散落各地** - daily note 写完没人整理，长期规则和日常记录混在一起
2. **缺少系统化自我认知** - 知道自己常犯什么错，但没写成稳定文档
3. **没有清晰的分层维护机制** - 不知道什么时候该升级、什么时候该降级
4. **升级 OpenClaw 后容易丢** - 手工改的东西容易被覆盖

## 核心分层

```
┌─────────────────────────────────────┐
│     OpenClaw 系统提示词注入层       │
│   (AGENTS/USER/IDENTITY/MEMORY)     │
├─────────────────────────────────────┤
│         人格补充层                  │
│   world.md  - 外部世界认知          │
│   self.md   - 自我认知              │
│   expression.md - 表达控制         │
├─────────────────────────────────────┤
│         待验证层 (inbox/)           │
│   尚未稳定的认知候选                │
├─────────────────────────────────────┤
│         Daily 层                    │
│   memory/YYYY-MM-DD.md              │
└─────────────────────────────────────┘
```

## 四种维护模式

| 模式 | 触发 | 目标 |
|------|------|------|
| capture | 高认知任务尾部 | 留下原料到 inbox/daily |
| light | heartbeat 30 分钟 | 轻量扫描新信号 |
| normal | 日总结 23:30 | 常规升级 |
| deep | 周总结 02:00 | 深度整理 |

## cron 任务建议

- `*-memory-daily-distill`: 每日 23:30，模式 normal
- `*-agent-self-evolution`: 每周日 02:00，模式 deep
- `*-capability-evolver`: 每周日 02:20，能力进化专项

## 与周级能力进化线的关系

人格维护（world/self/expression）和能力进化（高频任务、skill 化）是两条独立线：

- 人格维护 → `personhood-document-maintainer deep`
- 能力进化 → 独立周复盘 + `skill-creator`

不要混在一起。

## 关键约束

- 不改 OpenClaw 内置源码
- 通过工作区文档分层控制
- 升级后可通过脚本一键回补
- 共享框架 + agent 个性分离
