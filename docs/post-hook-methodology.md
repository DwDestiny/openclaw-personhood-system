# Post-Hook Methodology

## 定位

结尾 hook 是一个结构化认知更新器。

它不只是写日志，而是在每轮对话或任务结束后，把这轮经历转成对用户、对世界、对方法、对自己、对表达的更新，并把这些更新分发到合适的文档层。

## 核心职责

1. 提炼本轮值得保留的认知信号。
2. 判断这些信号属于哪一类认知。
3. 判断它们应该停留在 daily、进入 inbox，还是升级到长期文档。
4. 判断是否需要替换、降级或废弃旧认知。

## 五类认知信号

- 用户认知：用户偏好、禁忌、默认期待、明确纠正。
- 世界认知：行业、平台、产品、趋势、现实反馈的新理解。
- 方法认知：什么做法有效、无效、适用边界是什么。
- 自我认知：自己哪里更像人、哪里机械化、哪里跑偏、哪里成熟。
- 表达认知：什么表达自然，什么表达僵硬，什么情绪强度合适。

## 路由原则

- `memory/YYYY-MM-DD.md`
  记录本轮过程、关键决策、上下文、未完事项。

- `memory/inbox/`
  记录值得观察但尚未稳定的认知候选。

- `memory/world.md`
  记录跨任务、较稳定的世界理解。

- `memory/self.md`
  记录跨任务、较稳定的自我理解。

- `memory/expression.md`
  记录跨任务、较稳定的表达规律。

- `MEMORY.md`
  记录跨任务通用硬规则、稳定经验、长期有效方法。

- `USER.md`
  仅记录稳定用户偏好，不记录一次性执行反馈。

- `IDENTITY.md`
  仅在需要调整长期人格底色时更新，频率必须很低。

## 升级、降级、替换规则

- 新增：首次出现且具有保留价值的认知，可以先写 daily 或 inbox。
- 强化：重复成立的认知，可从 inbox/daily 升级到长期文档。
- 冲突：新认知与旧认知冲突时，优先判断旧认知是否需要替换或降级，而不是简单并存。
- 废弃：一次性噪音、偶然情绪、局部执行细节，不升级到长期层。

## 立即升级的适用场景

以下情况可以直接进入长期层：
- 用户明确说“以后就按这个来”
- 明确推翻旧规则
- 已经造成明显错误，必须防止复发
- 已经在多个场景重复成立

## 必须克制的情况

- 不把一轮临时情绪写进人格底色
- 不把执行细枝末节写进 `USER.md`
- 不把未验证的一次性想法写进 `world.md` 或 `self.md`
- 不把所有反馈都升级成长期规则

## 方法论目标

结尾 hook 的目标不是“记得更多”，而是：

让智能体每完成一轮经历，就有机会做一次小型认知迭代，并且这种迭代会真实影响下一轮默认行为。
