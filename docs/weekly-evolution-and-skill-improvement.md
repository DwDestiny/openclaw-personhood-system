# Weekly Evolution And Skill Improvement

## 目标

这份文档定义周复盘里除人格维护、认知维护之外的另一条独立职责：
- 发现高频重复任务
- 识别缺失 workflow / 缺失 skill
- 识别已有 skill 的执行偏差与高频失败点
- 必要时通过 `skill-creator` 创建新 skill 或优化已有 skill

这条线不等于人格化维护，也不应和 `world/self/expression` 文档维护混成一件事。

## 两条周复盘线

### 线 1：人格与认知维护

负责：
- `world/self/expression`
- `MEMORY.md`
- 必要时 `USER.md` / `IDENTITY.md`

入口：
- `personhood-document-maintainer deep`

### 线 2：能力与 skill 进化

负责：
- 回看最近一周高频任务
- 判断哪些任务已经稳定到值得 skill 化
- 判断哪些已有 skill 经常调用偏航、边界不清、执行质量不稳
- 通过 `skill-creator` 创建或优化 skill

入口：
- 每周专项复盘

## 周复盘要看什么

### 高频重复任务

重点看：
- 一周内是否多次出现同类任务
- 是否每次都要重新想流程
- 是否总在同一处出错或返工
- 是否值得沉淀成可复用 skill / workflow

### 已有 skill 偏差

重点看：
- 调用了 skill，但执行仍频繁跑偏
- skill 描述触发不准
- skill 边界不清，容易串步骤
- skill 缺少关键方法论或关键约束
- skill 过旧，和现在的稳定做法已经不一致

## 周复盘后的动作

默认顺序：
1. 先看这一周哪些任务反复出现。
2. 再看这些任务是否已经值得 workflow 化或 skill 化。
3. 再看已有 skill 是否存在系统性偏差，而不是单次失误。
4. 最后再决定是进入 `skill-creator`，还是只先记录为待观察问题。

### 若发现缺 skill

- 不要只把经验写进长期文档
- 应进入 `skill-creator` 流程
- 产出一个更完整、可复用、可迁移的 skill 草案或正式 skill

### 若发现 skill 有偏差

- 不要只在 daily 里吐槽
- 应进入 `skill-creator` 流程
- 优化 skill 的描述、边界、步骤、方法论、触发词或参考文档

### 若只是一次性例外

- 不进入 skill 化
- 先记入 daily 或 inbox，继续观察

## 边界提醒

- 周复盘里“人格维护”和“skill 进化”是两条线，不要混写
- 不是所有高频任务都要 skill 化，只有重复、稳定、可复用、值得标准化的才做
- 不是所有 skill 问题都要立刻重写，先看是不是偶发噪音
- 一旦进入 skill 创建或优化，默认优先使用 `skill-creator`
